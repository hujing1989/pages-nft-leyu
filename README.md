# pages-nft-leyu

## 项目简介

`pages-nft-leyu` 是一个用于归档和发布多个独立 HTML 页面的仓库。  
本仓库不针对任何特定域名或网站，仅作为静态页面的集中存放与版本管理工具。

## 目录说明

```
pages-nft-leyu/
├── index.html          # 首页入口（可选）
├── pages/              # 存放各独立 HTML 页面
│   ├── page1.html
│   ├── page2.html
│   └── ...
├── assets/             # 公共资源文件（样式、脚本、图片等）
│   ├── css/
│   ├── js/
│   └── images/
└── README.md           # 本文件
```

- `pages/`：主要存放各个独立 HTML 页面，可根据需要按子目录组织。
- `assets/`：存放页面共用的资源文件，便于统一维护。

## 页面归档说明

本仓库中收录的 HTML 页面均为独立静态文件，相互之间无依赖关系。  
每个页面均可在浏览器中直接打开运行，无需后端支持。  
页面内容及功能由对应文件自行定义，仓库仅负责版本管理与发布。

## 维护说明

- 新增页面：在 `pages/` 目录下创建新的 HTML 文件，并确保引用资源路径正确。
- 修改页面：直接编辑对应文件，提交前建议在本地测试。
- 删除页面：移除对应文件并更新相关索引（如有）。
- 资源更新：修改 `assets/` 下文件时，注意检查所有引用页面的兼容性。

欢迎提交 Pull Request 或 Issue 进行讨论。  
本仓库不承诺提供技术支持或持续更新，请自行评估使用风险。

## 许可证

本仓库内容按需使用，无特定许可证约束。  
如有特殊版权要求，请在各页面文件内自行标注。
